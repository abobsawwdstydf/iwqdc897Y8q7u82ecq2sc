## 📋 ПОШАГОВАЯ ИНСТРУКЦИЯ

### ШАГ 1: Подготовка проекта

#### 1.1 Инициализация Git
```bash
git init
git add .
git commit -m "Initial commit"
```

#### 1.2 Push на GitHub
```bash
# Создайте репозиторий на https://github.com/new
git remote add origin https://github.com/ВАШ_НИК/nimbus.git
git branch -M main
git push -u origin main
```

---

### ШАГ 2: Создание базы данных на Render

1. Зайдите на https://render.com
2. Войдите через GitHub
3. **New + → PostgreSQL**

Заполните:
| Поле | Значение |
|------|----------|
| Name | `nimbus-db` |
| Database | `nimbus` |
| User | `nimbus` |
| Region | `Frankfurt` |
| Plan | `Free` |

4. Нажмите **Create Database**
5. **Скопируйте `External Database URL`** (выглядит так):
```
postgresql://nimbus:XXXXXXXXXX@dpg-XXXXXX.frankfurt-postgres.render.com/nimbus_wnbp
```

---

### ШАГ 3: Создание Web Service

1. **New + → Web Service**
2. **Connect a repository** → Выберите ваш репозиторий

Заполните настройки:

| Поле | Значение |
|------|----------|
| **Name** | `nimbus-messenger` |
| **Region** | `Frankfurt` |
| **Branch** | `main` |
| **Root Directory** | `apps/server` |
| **Runtime** | `Node` |
| **Build Command** | `npm install && npm run build` |
| **Start Command** | `node dist/index.js` |
| **Plan** | `Free` |

---

### ШАГ 4: Переменные окружения

В разделе **Environment** добавьте:

| Key | Value |
|-----|-------|
| `NODE_ENV` | `production` |
| `DATABASE_URL` | `<URL из Шага 2>` |
| `JWT_SECRET` | `36427af4278b7198dc850c7235c4c85feda7275d89fe3d360c79a1af94579765` |
| `ENCRYPTION_KEY` | `36427af4278b7198dc850c7235c4c85feda7275d89fe3d360c79a1af94579765` |
| `CORS_ORIGINS` | `*` |
| `MAX_REGISTRATIONS_PER_IP` | `5` |
| `PORT` | `3001` |

---

### ШАГ 5: Деплой

1. Нажмите **Create Web Service**
2. Дождитесь завершения (5-10 минут)
3. Откройте **Logs** для мониторинга

---

### ШАГ 6: Инициализация БД

1. Откройте **Shell** в дашборде сервиса
2. Выполните команду:

```bash
npx prisma db push --force-reset --accept-data-loss
```

---

### ШАГ 7: Проверка

Откройте в браузере:
```
https://nimbus-messenger.onrender.com/api/health
```

Должно вернуться: `OK`

## 🆘 ТРАБЛШУТИНГ

### Ошибка: `@prisma/client did not initialize`
```bash
cd apps/server
npx prisma generate
```

### Ошибка: `Connection refused`
- Проверьте `DATABASE_URL` в Environment
- Убедитесь, что БД создана

### Ошибка: `Migration failed`
```bash
# В Shell на Render:
npx prisma db push --force-reset --accept-data-loss
```

### Сервис уходит в сон
- Free план засыпает через 15 мин без активности
- Решение: Pro план или ping-сервис (cron-job.org)
