# 📱 Настройка для доступа с телефона

## Проблема
При доступе к мессенджеру с телефона по локальной сети (например, `http://192.168.0.100:5173`) возникают ошибки:
- WebSocket connection failed
- getUserMedia не работает

## Решение

### 1️⃣ Настройка сервера

Сервер уже настроен на приём подключений из локальной сети!

**CORS разрешает:**
- ✅ `localhost:*` — все локальные порты
- ✅ `192.168.x.x` — локальная сеть
- ✅ `10.x.x.x` — корпоративная сеть
- ✅ `172.16-31.x.x` — Docker сети
- ✅ В режиме разработки — все origins

### 2️⃣ Настройка фронтенда

Создайте файл `apps/web/.env`:

```env
# Укажите IP вашего компьютера и порт сервера
VITE_API_URL=http://192.168.0.100:3001
```

**Важно:**
- IP должен быть статическим или одинаковым в вашей сети
- Порт по умолчанию: `3001`
- Не используйте `localhost` для доступа с других устройств

### 3️⃣ Запуск сервера

```bash
# В терминале 1 — запускаем сервер
cd apps/server
npm run dev
```

Сервер будет доступен по:
- `http://localhost:3001` — на компьютере
- `http://192.168.0.100:3001` — в локальной сети

### 4️⃣ Запуск фронтенда

```bash
# В терминале 2 — запускаем фронтенд
cd apps/web
npm run dev -- --host
```

**Важно:** Флаг `--host` открывает доступ к Vite серверу из сети!

Фронтенд будет доступен по:
- `http://localhost:5173` — на компьютере
- `http://192.168.0.100:5173` — в локальной сети

### 5️⃣ Доступ с телефона

1. Убедитесь что телефон и компьютер в одной Wi-Fi сети
2. Откройте браузер на телефоне
3. Перейдите на: `http://192.168.0.100:5173`
4. Войдите в аккаунт

## ⚠️ Ошибка getUserMedia / Media devices not available

**Проблема:** Браузеры требуют HTTPS для доступа к микрофону/камере, кроме localhost.

**Решения:**

### Вариант 1: Использовать localhost (только Android + Chrome)
В Chrome на Android можно открыть `chrome://flags` и включить:
- `#unsafely-treat-insecure-origin-as-secure`
- Добавьте ваш IP: `http://192.168.0.100:5173`

### Вариант 2: HTTPS через ngrok (рекомендуется)
```bash
# Установите ngrok
npm install -g ngrok

# Запустите туннель
ngrok http 5173
```

Получите HTTPS URL: `https://xxxx-xxxx.ngrok.io`

### Вариант 3: HTTPS через mkcert (для разработки)
```bash
# Установите mkcert
npm install -g mkcert

# Создайте сертификат
mkcert -install
mkcert localhost 192.168.0.100

# Запустите Vite с HTTPS
npm run dev -- --host --https
```

## 🔧 Проверка подключения

### Сервер запущен?
```bash
curl http://localhost:3001/api/auth/me
# Должен вернуть 401 (нет токена) — это нормально
```

### Фронтенд доступен?
Откройте в браузере:
- Компьютер: `http://localhost:5173`
- Телефон: `http://192.168.0.100:5173`

### WebSocket работает?
Откройте консоль браузера (F12) и проверьте что нет ошибок WebSocket.

---

## 📊 Типичные проблемы

| Проблема | Решение |
|----------|---------|
| `WebSocket connection failed` | Проверьте что сервер запущен и VITE_API_URL правильный |
| `Media devices not available` | Используйте HTTPS или localhost |
| `404 Not Found` | Проверьте пути API в `.env` |
| `CORS error` | Сервер должен быть запущен с правильным CORS |

## 🎯 Быстрая настройка для телефона

1. **Узнайте IP компьютера:**
   - Windows: `ipconfig` → ищите `IPv4 Address`
   - macOS/Linux: `ifconfig` → ищите `inet`

2. **Создайте `.env` для фронтенда:**
   ```env
   VITE_API_URL=http://ВАШ_IP:3001
   ```

3. **Запустите сервер:**
   ```bash
   npm run dev -w apps/server
   ```

4. **Запустите фронтенд с --host:**
   ```bash
   npm run dev -w apps/web -- --host
   ```

5. **Откройте на телефоне:**
   ```
   http://ВАШ_IP:5173
   ```

---

**Готово!** 🎉 Мессенджер должен работать с телефона!
