## 🚀 Быстрый старт

### Локальная разработка:

```bash
# 1. Установка
npm install

# 2. Настройка .env
cp .env.example .env
# Отредактируйте .env (DATABASE_URL, JWT_SECRET, etc.)

# 3. Настройка .storage.env для хранилища
cp .storage.env.example .storage.env
# Добавьте Telegram боты, каналы, Discord вебхуки

# 4. Запуск
npm run dev
```

### Деплой на Render:

1. **Создай PostgreSQL** для мессенджера
2. **Создай PostgreSQL** для хранилища
3. **Добавь переменные** из `.env` и `.storage.env`
4. **Build Command:**
   ```bash
   npm install && npm run build
   ```
5. **Start Command:**
   ```bash
   node apps/server/dist/index.js
   ```

## 📦 Распределённое хранилище

### Архитектура:

```
Файл (до 50GB)
    ↓
Разбиение на чанки (19MB каждый)
    ↓
Шифрование (случайный метод из 20)
    ↓
Распределение:
  - Telegram канал 1: чанки 1-35
  - Discord сервер 1: чанки 36-70
  - Telegram канал 2: чанки 71-105
    ↓
Метаданные в БД (ключи, IV, теги)
```

### Настройка:

1. **Создай Telegram ботов** (@BotFather)
2. **Создай каналы** и добавь ботов админами
3. **Узнай ID каналов** (@RawDataBot)
4. **Создай Discord вебхуки** (настройки сервера)
5. **Заполни `.storage.env`**:
   ```env
   TELEGRAM_BOT_TOKENS=token1,token2,token3
   TELEGRAM_CHANNEL_IDS=-1001234567890,-1001234567891
   DISCORD_WEBHOOK_URLS=https://discord.com/...
   STORAGE_DATABASE_URL=postgresql://...
   ```

### API хранилища:

```http
POST   /api/storage/upload           # Загрузить файл
GET    /api/storage/download/:id     # Скачать файл
DELETE /api/storage/delete/:id       # Удалить файл
GET    /api/storage/info/:id         # Информация
GET    /api/storage/health           # Статус ресурсов
```

## 🌐 Универсальный веб-клиент

### Подключение к серверу на Render:

Создай `.env` в `apps/web/`:

```env
VITE_API_URL=https://nimbus-msg.onrender.com
VITE_STORAGE_URL=https://nimbus-msg.onrender.com
```

### Запуск локального веб-клиента:

```bash
cd apps/web
npm run dev
```

Веб-клиент подключится к серверу на Render!

### Свой веб-клиент:

```typescript
import { createNimbusClient } from './lib/nimbus-client';

const client = createNimbusClient(
  'https://nimbus-msg.onrender.com',  // Сервер
  'https://nimbus-msg.onrender.com'   // Хранилище (опционально)
);

// Шифрование сообщения
await client.generateKey('my-secret-password');
const encrypted = await client.encryptMessage('Hello!');

// Загрузка файла
const result = await client.uploadFile(file);
```

## 🔐 Безопасность

### Шифрование:

1. **На клиенте:**
   - AES-256-GCM
   - Ключ из пароля (PBKDF2, 100000 итераций)
   - Сообщение шифруется ДО отправки

2. **На сервере:**
   - 20 методов (AES, ChaCha20, ARIA, Camellia)
   - Случайный выбор на каждый чанк
   - Уникальный ключ на чанк (32 байта)
   - Ключ шифруется мастер-ключом

### Протокол общения:

```
Клиент                          Сервер
  │                              │
  ├──[Зашифрованное сообщение]──>│
  │   (AES-256-GCM, клиент)      │
  │                              │
  │                              ├──[В хранилище]──> Telegram/Discord
  │                              │   (20 методов, сервер)
  │                              │
  │<──[Подтверждение]────────────┤
  │                              │
```

### Защита файлов:

- ✅ Файлы **не удаляются** (псевдо-удаление для пользователей)
- ✅ **50GB лимит** на сервере
- ✅ **20GB лимит** для пользователей
- ✅ **Дублирование** в Telegram + Discord
- ✅ **Шифрование** каждого чанка
