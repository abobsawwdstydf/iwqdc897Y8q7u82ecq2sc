## ⚡ БЫСТРЫЙ СТАРТ (5 МИНУТ)

### Шаг 1: Подготовьте проект

Убедитесь что у вас есть:
- ✅ Файл `.render.yaml` в корне проекта
- ✅ Все файлы проекта в одной папке
- ✅ Git репозиторий на GitHub

### Шаг 2: Загрузите на GitHub

```bash
# В папке проекта:
git init
git add .
git commit -m "Deploy to Render"
git branch -M main
git remote add origin https://github.com/ВАШ_USERNAME/ВАШ_REPO.git
git push -u origin main
```

### Шаг 3: Создайте проект на Render

1. Перейдите на https://render.com
2. Войдите через GitHub (кнопка "Sign in with GitHub")
3. Нажмите "New +" → "Blueprint"
4. Выберите ваш репозиторий
5. Нажмите "Connect"

### Шаг 4: Примените конфигурацию

1. Render автоматически найдёт файл `.render.yaml`
2. Проверьте настройки:
   - **Name**: `nimbus-messenger`
   - **Region**: Frankfurt (или ближайший)
   - **Branch**: `main`
   - **Plan**: Free
3. Нажмите "Apply"

### Шаг 5: Дождитесь деплоя

1. Сборка займёт 3-5 минут
2. Проверьте логи: Dashboard → nimbus-messenger → Logs
3. Когда статус будет "Live" — готово!

## 🛠️ ЕСЛИ ЧТО-ТО ПОШЛО НЕ ТАК:

### ❌ Ошибка: "Build failed"

**Проверьте логи:**
```
Dashboard → nimbus-messenger → Logs → Build Logs
```

**Частые проблемы:**
1. **Нет package.json** — проверьте структуру
2. **Ошибки TypeScript** — запустите `npm run build` локально
3. **Нет .render.yaml** — создайте файл в корне

### ❌ Ошибка: "Database connection failed"

**Решение:**
1. Проверьте что БД создана
2. Проверьте DATABASE_URL в настройках
3. Запустите миграции в Shell:
```bash
cd apps/server
npx prisma migrate deploy
```

### ❌ Ошибка: "Port already in use"

**Решение:**
Убедитесь что в `apps/server/src/index.ts`:
```typescript
const port = process.env.PORT || 3001;
```

### ❌ Ошибка: "CORS error"

**Решение:**
1. Проверьте CORS_ORIGINS
2. Убедитесь что URL совпадает с вашим доменом
3. Добавьте https:// префикс
