# 🚀 Nexo Messenger - Полная инструкция

## 📋 Что такое Nexo?

Nexo - это мессенджер с **децентрализованным хранением файлов** через Telegram и Discord.

### Особенности:
- ✅ Файлы хранятся в Telegram/Discord (не на сервере!)
- ✅ 20 методов шифрования (AES, ChaCha20, Aria, Camellia)
- ✅ Автоматическое распределение нагрузки
- ✅ Звонки WebRTC (голос/видео)
- ✅ Папки чатов как в Telegram
- ✅ Видео-кружки
- ✅ Адаптация под мобильные

## 🔧 Установка

### 1. Клонирование и установка зависимостей

```bash
# Перейди в папку проекта
cd telega

# Установи все зависимости
npm install
```

### 2. Настройка .env

Файл `.env` уже настроен! Проверь:

```bash
# Открой .env и убедись, что все поля заполнены
notepad .env
```

**Важные поля:**
- `DATABASE_URL` - PostgreSQL (уже есть от Render)
- `TELEGRAM_BOT_TOKENS` - токены ботов (уже есть)
- `TELEGRAM_CHANNEL_IDS` - ID каналов (уже есть)
- `DISCORD_WEBHOOK_URLS` - вебхуки Discord (уже есть)
- `MASTER_KEY` - ключ шифрования (уже есть)

### 3. Установка Redis

**Для Windows:**
1. Скачай Redis для Windows: https://github.com/microsoftarchive/redis/releases
2. Установи или используй Docker:
   ```bash
   docker run -d -p 6379:6379 redis:latest
   ```

**Для Render:**
Redis автоматически подключится через `REDIS_URL` (если нет - используется локальный)

### 4. Первый запуск

```bash
# Установка БД
npm run db:setup

# Запуск (сервер + фронтенд + хранилище)
npm run dev:full
```

Или просто:
```bash
npm run dev
```

## 🌐 Запуск на Render

### 1. Создай PostgreSQL на Render

1. Зайди на https://render.com
2. New+ → **PostgreSQL**
3. Name: `nexo-db`
4. Region: `Frankfurt`
5. Скопируй **External Database URL**

### 2. Создай Web Service

1. New+ → **Web Service**
2. Подключи репозиторий GitHub
3. Выбери: `abobsawwdstydf/iwqdc897Y8q7u82ecq2sc`

### 3. Настройки

```
Name: nexo-messenger
Region: Frankfurt
Branch: main
Root Directory: (оставь пустым)
Runtime: Node
Build Command: npm install && npm run build
Start Command: npm run dev
```

### 4. Environment Variables

Добавь все переменные из `.env`:

```
NODE_ENV=production
PORT=3001
DATABASE_URL=postgresql://...
JWT_SECRET=...
ENCRYPTION_KEY=...
MASTER_KEY=...
CORS_ORIGINS=*
TELEGRAM_BOT_TOKENS=...
TELEGRAM_CHANNEL_IDS=...
DISCORD_WEBHOOK_URLS=...
ADMIN_TOKEN=qwertyuiopasd
```

### 5. Redis (опционально)

Для продакшена создай Redis на Render:
1. New+ → **Redis**
2. Скопируй URL
3. Добавь в ENV: `REDIS_URL=redis://...`

## 📱 Telegram - настройка

### 1. Получение токена бота

1. Открой @BotFather в Telegram
2. Отправь `/newbot`
3. Придумай имя (например: `Nexo Storage Bot`)
4. Придумай username (должен заканчиваться на `bot`: `nexo_storage_bot`)
5. Скопируй токен (например: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)

### 2. Создание канала

1. Создай новый канал в Telegram
2. Сделай бота **администратором** канала
3. Добавь @RawDataBot в канал
4. Скопируй `id` из сообщения (например: `-1001234567890`)
5. Удали @RawDataBot

### 3. Добавление в .env

```env
TELEGRAM_BOT_TOKENS=токен1,токен2,токен3
TELEGRAM_CHANNEL_IDS=-1001234567890,-1002345678901
```

## 💬 Discord - настройка

### 1. Создание вебхука

1. Открой Discord
2. Зайди на сервер → настройки канала (⚙️)
3. Интеграции → Вебхуки
4. Создать вебхук
5. Скопируй URL вебхука

### 2. Добавление в .env

```env
DISCORD_WEBHOOK_URLS=https://discord.com/api/webhooks/123/abc,https://discord.com/api/webhooks/456/def
```

## 📊 API

### Загрузка файла

```bash
POST /upload
Content-Type: multipart/form-data

{
  "file": <файл>
}

Ответ:
{
  "fileId": "uuid",
  "chunks": 5,
  "status": "uploading"
}
```

### Скачивание файла

```bash
GET /download/:fileId
```

### Статус файла

```bash
GET /status/:fileId

Ответ:
{
  "fileId": "uuid",
  "status": "ready",
  "name": "file.txt",
  "size": 1024,
  "chunks": 5
}
```

## 🛡️ Безопасность

- ✅ AES-256-GCM шифрование
- ✅ 20 различных методов шифрования
- ✅ Ключи шифруются мастер-ключом
- ✅ Файлы разбиваются на чанки по 19MB
- ✅ Распределение по разным ботам/вебхукам

---

## ⚠️ Важно

1. **Redis обязателен** для работы очереди
2. **Минимум 1 Telegram бот** и **1 канал** для хранения
3. **Не удаляй файлы из Telegram/Discord** - они зашифрованы и разбиты на чанки
4. **MASTER_KEY** должен быть одинаковым на всех серверах

## 🆘 Troubleshooting

### Ошибка "No resources available"
- Проверь, что боты добавлены в каналы
- Проверь токены ботов в .env
- Проверь, что боты - администраторы каналов

### Ошибка подключения к БД
- Проверь DATABASE_URL
- Убедись, что PostgreSQL доступен
- Для Render: используй External Database URL

### Ошибка Redis
- Установи Redis: `docker run -d -p 6379:6379 redis:latest`
- Или добавь REDIS_URL в .env

## 📞 Звонки

Звонки работают через WebRTC:

1. **Голосовые звонки** - микрофон
2. **Видео звонки** - камера + микрофон
3. **Групповые звонки** - до 10 участников

### Настройка STUN/TURN

Для работы звонков через NAT нужен TURN сервер:

```env
STUN_URLS=stun:stun.l.google.com:19302
TURN_URL=turn:your-server.com:3478
TURN_SECRET=your_secret
```

Или используй бесплатный Coturn:
```bash
# Установка на Ubuntu
apt install coturn

# Настройка
vi /etc/turnserver.conf
```
